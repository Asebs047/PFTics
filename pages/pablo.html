Documentación GlassFish Server

1. ¿Qué es GlassFish?

GlassFish es un servidor de aplicaciones open source inicialmente desarrollado por Sun Microsystems y actualmente mantenido por la Eclipse Foundation. Permite desplegar y administrar aplicaciones empresariales basadas en Java EE o Jakarta EE, facilitando el manejo de componentes como servlets, EJB, y JSP en un entorno robusto y escalable.

2. ¿Para qué sirve?

GlassFish sirve para ejecutar y administrar aplicaciones web y empresariales desarrolladas en Java, proporcionando servicios esenciales como manejo de transacciones, seguridad, conectividad a bases de datos y escalabilidad, que son necesarios para aplicaciones robustas en ambientes productivos y de desarrollo.

3. Ventajas y Desventajas

Ventajas principales:

  • Es gratuito y open source, con amplio soporte de la comunidad.
  • Compatible con estándares Java EE/Jakarta EE, lo que facilita portabilidad.
  • Fácil de usar y configurar para desarrolladores.

Desventajas:

  • Consumo relativamente alto de recursos en comparación con otros servidores.
  • Soporte comercial menos robusto que servidores propietarios.

4. ¿Cómo se aplica?

GlassFish se utiliza para desplegar aplicaciones empresariales Java, facilitando la gestión de servicios como administración de sesiones, seguridad, conexión a bases de datos y escalabilidad. Es común en ambientes de desarrollo y producción donde se requiere un servidor confiable y compatible con Java EE/Jakarta EE.

5. Ejemplos

Algunas aplicaciones típicas que pueden correr en GlassFish incluyen sistemas de comercio electrónico, portales web corporativos, aplicaciones de gestión empresarial y servicios basados en microservicios que requieren un entorno compatible con Java EE/Jakarta EE.

6. Arquitectura de GlassFish

La arquitectura de GlassFish está basada en un modelo modular y distribuido que incluye dominios, clusters, nodos y servidores. Esto permite escalar aplicaciones y administrar recursos de forma eficiente, facilitando el despliegue y gestión en entornos locales o en la nube.

7. Compatibilidad con tecnologías Java EE / Jakarta EE

GlassFish es totalmente compatible con las especificaciones completas de Java EE y Jakarta EE, soportando tecnologías como servlets, JSP, EJB, JPA, JAX-RS, entre otras, lo que asegura que las aplicaciones desarrolladas con estas tecnologías puedan ejecutarse y escalar adecuadamente.

8. Administración y monitoreo

GlassFish incluye herramientas integradas para la administración y monitoreo del servidor y las aplicaciones, como una consola web y soporte para JMX. Esto permite a los administradores configurar, supervisar el rendimiento y mantener la salud del entorno de manera eficiente.
